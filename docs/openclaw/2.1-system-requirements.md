## Runtime Requirements

### Node.js Version

OpenClaw requires **Node.js 22 or later**. This is enforced throughout the codebase and installer scripts.

**Primary runtime:** Node.js is the recommended and tested runtime for the Gateway. Bun support exists for development workflows but is explicitly **not recommended** for Gateway production use due to compatibility issues with WhatsApp (Baileys) and Telegram providers.

## Hardware Requirements

### Memory (RAM)

```
Add swapLarge contextMinimum
512MB-1GB
Raspberry Pi baselineHeavy workload
4GB+
Local models + concurrent
```

**Minimum:** 512MB-1GB RAM sufficient for personal use on Raspberry Pi.

**Recommended:** 2GB RAM for production VPS deployments. This prevents OOM issues under moderate load.

**Heavy workloads:** 4GB+ for local model inference, high concurrency, or verbose logging.

**VPS examples:**

- Fly.io: 512MB often causes OOM; 2GB (`memory = "2048mb"`) recommended
- DigitalOcean: $6/mo tier (1GB) may require swap
- Oracle Cloud: Always Free tier provides up to 24GB on ARM

**Code entities:**

- Fly.io memory config: [fly.toml 30](https://github.com/openclaw/openclaw/blob/bf6ec64f/fly.toml#L30-L30)
- Node.js memory flag: `NODE_OPTIONS = "--max-old-space-size=1536"` [fly.toml 15](https://github.com/openclaw/openclaw/blob/bf6ec64f/fly.toml#L15-L15)

### Storage

**Minimum:** ~500MB for CLI + dependencies + Gateway runtime.

**Recommended:** 5-20GB to accommodate:

- Session history + transcripts
- Memory search indices
- Media files (images, audio clips)
- Docker images (if using containerized deployment)
- Logs

**VPS defaults:**

- Fly.io: Persistent volume size configurable (1GB typical)
- Oracle Cloud: Up to 200GB block volume (Always Free)
- GCP: 20GB boot disk typical

**State directory structure:**

```
~/.openclaw/
├── openclaw.json          # Main config
├── credentials/           # Auth profiles, OAuth tokens
├── agents/<agentId>/
│   ├── sessions/         # Session transcripts (JSONL)
│   └── memory/           # Embeddings, BM25 indices
└── workspace/            # User files, skills, memory
```

### CPU

**Minimum:** 1 core sufficient for single-user, API-based models.

**Recommended:** 2+ cores for:

- Concurrent agent processing (`agents.defaults.maxConcurrent`)
- Local model inference
- Media processing (image uploads, audio transcription)

**Architecture:**

- x86\_64: Primary support
- ARM64 (aarch64): Supported, tested on Raspberry Pi and Oracle Cloud
- ARM32: Not recommended

**Code entities:**

### Firewall Considerations

**Loopback binding (default):** Gateway binds to `127.0.0.1` and requires SSH tunnel or Tailscale Serve for remote access.

**LAN binding (`--bind lan`):** Binds to `0.0.0.0` and requires `gateway.auth.token` or `gateway.auth.password` for security.

**Tailnet binding (`--bind tailnet`):** Binds to Tailscale IP; requires Tailscale installed and authenticated.

**Bonjour/mDNS (local discovery):**

- Service type: `_openclaw-gw._tcp`
- Used by iOS/Android apps to discover local gateways

**DNS-SD (wide-area discovery):**

## Pre-Installation Checklist

Before installing OpenClaw, verify:

**Core requirements:**

- Node.js 22 or later installed (`node -v`)
- npm or pnpm available
- `git` available (required for `--install-method git`)

**Platform-specific:**

- macOS: Xcode Command Line Tools (if building from source)
- Linux: systemd enabled (`systemctl --user status`)
- Windows: WSL2 installed and systemd enabled in `/etc/wsl.conf`

**Network:**

- Port 18789 available (or alternative port configured)
- Firewall rules allow loopback/lan/tailnet binding as needed

**Storage:**

- At least 5GB free disk space
- Write permissions in `~/.openclaw/` (or `$OPENCLAW_STATE_DIR`)

**Optional:**

- Docker and Docker Compose (if using containerized deployment)
- Tailscale installed (if using Tailscale for remote access)
- Build tooling (gcc, make, node-gyp) for native module compilation
