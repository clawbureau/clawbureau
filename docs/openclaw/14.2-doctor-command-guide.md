## Purpose and Scope

This page documents the `openclaw doctor` command, OpenClaw's automated diagnostic and repair tool. Doctor performs configuration migrations, health checks, state integrity verification, service audits, and automated repairs across the entire OpenClaw system. It is the primary tool for fixing stale configurations, migrating legacy formats, and diagnosing runtime issues.

For general troubleshooting workflows, see [Common Issues](https://deepwiki.com/openclaw/openclaw/14.3-common-issues). For health monitoring during normal operations, see [Health Monitoring](https://deepwiki.com/openclaw/openclaw/14.1-health-monitoring). For manual configuration management, see [Configuration Commands](https://deepwiki.com/openclaw/openclaw/12.5-configuration-commands).

## Command Overview

The doctor command provides automated diagnosis and repair with multiple run modes:

| Mode | Behavior |
| --- | --- |
| `openclaw doctor` | Interactive mode with prompts for each repair action |
| `openclaw doctor --yes` | Accepts all repair defaults without prompting |
| `openclaw doctor --repair` | Applies recommended repairs automatically |
| `openclaw doctor --non-interactive` | Runs safe migrations only, skips interactive repairs |
| `openclaw doctor --deep` | Scans system services for extra gateway installations |
| `openclaw doctor --fix` | Alias for `--repair` |

## Workspace Status Checks

Doctor inspects the workspace directory structure and suggests improvements:

| Check | Condition | Action |
| --- | --- | --- |
| **MEMORY.md exists** | File present | No action |
| **MEMORY.md missing** | `shouldSuggestMemorySystem()` | Suggest creating MEMORY.md for context |
| **Extra workspace dirs** | `~/openclaw` detected | Warn about multiple workspace locations |
| **Skills directory** | No skills found | Suggest running `openclaw configure --section skills` |
| **Bootstrap files** | Missing AGENTS.md/SOUL.md/TOOLS.md | Silent (bootstrap runs on next agent start) |

**MEMORY.md Suggestion Prompt**:

```
Workspace tip: Create MEMORY.md in your workspace for persistent
context that agents can reference across sessions.

Example:
echo "# Project Context" > ~/.openclaw/workspace/MEMORY.md
echo "- Project: My Application" >> ~/.openclaw/workspace/MEMORY.md
echo "- Tech Stack: Node.js, TypeScript" >> ~/.openclaw/workspace/MEMORY.md
```

## Output and Summary

At the end of execution, doctor writes a summary showing:

1. **Config changes applied**: Lists migrations, repairs, and new settings
2. **Warnings issued**: Security concerns, missing files, service issues
3. **Repair recommendations**: Actions requiring manual intervention
4. **Backup location**: Path to `openclaw.json.bak` if config was modified

Doctor updates the config with wizard metadata to track the last doctor run:

```
{
  wizard: {
    lastRunAt: "2025-01-14T10:30:00Z",
    lastRunVersion: "1.2.3",
    lastRunCommit: "abc123",
    lastRunCommand: "doctor",
    lastRunMode: "local"
  }
}
```
