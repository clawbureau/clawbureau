## Core CLI and Gateway Build

### Step 1: Clone Repository

```
git clone https://github.com/openclaw/openclaw.git
cd openclaw
```

### Step 2: Install Dependencies

```
pnpm install
```

This command:

- Installs root workspace dependencies
- Installs plugin dependencies (hoisted by pnpm workspace)
- Applies pnpm patches (if any exist in `pnpm.patchedDependencies`)

**Notes:**

- **Do not use `npm install`.** The workspace configuration requires pnpm.
- Bun is supported (`bun install`) but keep `pnpm-lock.yaml` synchronized

**Sources:**[AGENTS.md 40-42](https://github.com/openclaw/openclaw/blob/bf6ec64f/AGENTS.md#L40-L42) [docs/install/index.md 89-95](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/index.md#L89-L95)

### Step 3: Build Core

```
pnpm build
```

This executes the TypeScript compiler (`tsc`) and generates:

- [dist/](https://github.com/openclaw/openclaw/blob/bf6ec64f/dist/): Compiled JavaScript
- [dist/build-info.json](https://github.com/openclaw/openclaw/blob/bf6ec64f/dist/build-info.json): Version and commit metadata

The build script also runs [scripts/write-build-info.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/scripts/write-build-info.ts) to embed the current Git commit hash.

**Build configuration:**

- TypeScript config: [tsconfig.json](https://github.com/openclaw/openclaw/blob/bf6ec64f/tsconfig.json)
- Compiler version tracked in [package.json](https://github.com/openclaw/openclaw/blob/bf6ec64f/package.json)

**Sources:**[AGENTS.md 47](https://github.com/openclaw/openclaw/blob/bf6ec64f/AGENTS.md#L47-L47) [docs/reference/RELEASING.md 28](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/reference/RELEASING.md#L28-L28) [scripts/write-build-info.ts 1-49](https://github.com/openclaw/openclaw/blob/bf6ec64f/scripts/write-build-info.ts#L1-L49) [src/infra/git-commit.ts 1-110](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/infra/git-commit.ts#L1-L110)

### Step 4: Verify Build

```
# Check CLI entrypoint
node dist/index.js --version

# Or use the workspace script
pnpm openclaw --version
```

The CLI version is resolved from [package.json](https://github.com/openclaw/openclaw/blob/bf6ec64f/package.json) and the commit hash from [dist/build-info.json](https://github.com/openclaw/openclaw/blob/bf6ec64f/dist/build-info.json)

## Extension Builds

Extensions (plugins) live under [extensions/](https://github.com/openclaw/openclaw/blob/bf6ec64f/extensions/) as workspace packages. Each extension has its own `package.json`.

### Building Extensions

Extensions are **not** compiled during the core build. They are loaded at runtime via `jiti` (JIT TypeScript execution).

However, before publishing, extensions should be synchronized:

```
pnpm plugins:sync
```

This script:

- Aligns extension versions with the root `package.json`
- Updates extension changelogs

**Runtime dependency installation:**

When loading an extension, the Gateway runs:

```
npm install --omit=dev
```

in the extension directory. This means:

- Runtime dependencies must be in `dependencies` (not `devDependencies`)
- Avoid `workspace:*` protocol in `dependencies` (breaks `npm install`)

**Plugin SDK:**

The plugin SDK types are generated in [dist/plugin-sdk/](https://github.com/openclaw/openclaw/blob/bf6ec64f/dist/plugin-sdk/) during the core build.

**Sources:**[AGENTS.md 9-10](https://github.com/openclaw/openclaw/blob/bf6ec64f/AGENTS.md#L9-L10) [docs/reference/RELEASING.md 21](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/reference/RELEASING.md#L21-L21)

## Troubleshooting

### Build Failures

**Symptom:**`tsc` errors or missing types

**Fix:**

```
# Clean and rebuild
rm -rf dist node_modules
pnpm install
pnpm build
```

**Symptom:**`sharp` native build failures

**Fix:** Force prebuilt binaries:

```
SHARP_IGNORE_GLOBAL_LIBVIPS=1 pnpm install
```

**Sources:**[docs/install/index.md 68-74](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/index.md#L68-L74) [AGENTS.md 115](https://github.com/openclaw/openclaw/blob/bf6ec64f/AGENTS.md#L115-L115)

---

### UI Build Failures

**Symptom:**`pnpm ui:build` fails with missing dependencies

**Fix:**

```
# Reinstall UI dependencies
rm -rf ui/node_modules
pnpm ui:install
pnpm ui:build
```

**Sources:**[docs/install/index.md 94](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/index.md#L94-L94)

---

### Extension Build Issues

**Symptom:** Extension fails to load with `npm install` errors

**Fix:** Ensure runtime dependencies are in `dependencies`, not `devDependencies`. Avoid `workspace:*` protocol in `dependencies`.

**Sources:**[AGENTS.md 9-10](https://github.com/openclaw/openclaw/blob/bf6ec64f/AGENTS.md#L9-L10)

---

**Symptom:**`xcodebuild` fails with code signing errors

**Fix:** Use development signing for local builds:

```
BUILD_CONFIG=debug scripts/package-mac-app.sh
```

For release builds, ensure signing identity and provisioning profile are configured.

**Sources:**[docs/platforms/mac/release.md 1-78](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/platforms/mac/release.md#L1-L78)

---

### Test Failures

**Symptom:** Tests pass locally but fail in CI

**Check:**

- Environment variables (CI may not have API keys)
- Docker availability
- Network connectivity

**Symptom:** Coverage below threshold

**Fix:** Add tests or adjust thresholds in `vitest.config.ts`
