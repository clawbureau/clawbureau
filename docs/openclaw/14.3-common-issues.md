## Scope

This troubleshooting guide addresses:

- **Installation failures**: PATH issues, Node version mismatches, npm errors
- **Gateway connectivity**: port conflicts, auth failures, bind mode problems
- **Authentication issues**: OAuth expiry, API key resolution, profile rotation
- **Configuration errors**: schema validation, legacy migrations, missing keys
- **Channel problems**: WhatsApp login, Telegram rate limits, Discord auth
- **Model/provider failures**: rate limiting, credential resolution, fallback chains
- **Service management**: systemd/launchd issues, daemon restart loops
- **Sandbox/Docker**: image builds, container lifecycle

## Installation Issues

### openclaw: command not found

**Symptom**: After installation, the `openclaw` command is not recognized in new terminal sessions.

**Root cause**: npm global bin directory is not on PATH.

**Diagnosis**:

```
node -v          # Should be >= 22
npm -v
npm prefix -g    # Note the path
echo "$PATH"     # Check if $(npm prefix -g)/bin is present
```

**Fix paths by shell**:

| Shell | File | Add line |
| --- | --- | --- |
| zsh | `~/.zshrc` | `export PATH="$(npm prefix -g)/bin:$PATH"` |
| bash | `~/.bashrc` or `~/.bash_profile` | `export PATH="$(npm prefix -g)/bin:$PATH"` |
| fish | `~/.config/fish/config.fish` | `set -gx PATH (npm prefix -g)/bin $PATH` |

**Windows**: Ensure `%AppData%\npm` is on PATH (System Properties → Environment Variables).

**Sources**: [docs/install/index.md 154-180](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/index.md#L154-L180) [docs/help/faq.md 518-537](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L518-L537)

### Node version mismatch

**Symptom**: `openclaw` CLI exits with "Node version must be >= 22" or similar.

**Detection**:

```
node -v          # Check current
command -v node  # Check which binary
```

**Fix** (version manager approach):

```
# Install nvm (Linux/macOS)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
nvm install 22
nvm use 22
nvm alias default 22

# Reinstall openclaw
npm install -g openclaw@latest
```

**System Node check**: If using Homebrew (macOS), ensure Node 22+ is installed:

```
brew install node@22
brew link node@22 --force --overwrite
```

### Port 18789 already in use

**Symptom**: Gateway logs show `EADDRINUSE` or "port already in use".

**Diagnosis**:

```
# Check what's using the port
lsof -i :18789                    # macOS/Linux
netstat -ano | findstr :18789     # Windows

# Check if multiple gateway services exist
openclaw doctor --deep
```

**Fix options**:

1. **Kill conflicting process**:
	```
	lsof -ti:18789 | xargs kill -9
	openclaw gateway restart
	```
2. **Change gateway port**:
	```
	openclaw config set gateway.port 18790
	openclaw doctor --fix  # Updates service config
	openclaw gateway restart
	```
3. **Remove duplicate gateway services** (detected by `openclaw doctor --deep`):
	- macOS: `launchctl unload ~/Library/LaunchAgents/<label>.plist`
	- Linux: `systemctl --user disable --now <service>`

**Code path**: Port binding happens in [src/gateway/server.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/gateway/server.ts) via `resolveGatewayPort()` from [src/config/config.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/config/config.ts#LNaN-LNaN)

**Sources**: [docs/help/faq.md 160-168](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L160-L168) [src/commands/doctor-gateway-services.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/commands/doctor-gateway-services.ts)

### Gateway bind mode confusion

**Symptom**: Control UI shows "unauthorized" after changing `gateway.bind` to `lan` or `tailnet`.

**Bind modes and auth requirements**:

| Mode | Bind address | Requires token/password? | Use case |
| --- | --- | --- | --- |
| `loopback` | `127.0.0.1` | No (optional) | Local development |
| `lan` | `0.0.0.0` | **Yes** (required) | Local network access |
| `tailnet` | Tailscale IP | **Yes** (unless `allowTailscale: true`) | Tailscale mesh |
| `custom` | `gateway.customBindHost` | **Yes** | Specific interface |

**Fix**:

```
# Verify bind + auth config
openclaw config get gateway.bind
openclaw config get gateway.auth.mode
openclaw config get gateway.auth.token

# Generate token if missing
openclaw doctor --generate-gateway-token

# Or set manually
openclaw config set gateway.auth.token "$(openssl rand -hex 32)"
openclaw gateway restart
```

**Code path**: Auth resolution in [src/gateway/auth.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/gateway/auth.ts#LNaN-LNaN) bind address in [src/gateway/server.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/gateway/server.ts)

**Sources**: [docs/help/faq.md 95-96](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L95-L96) [src/commands/doctor.ts 122-158](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/commands/doctor.ts#L122-L158) [docs/gateway/authentication.md](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/authentication.md)

### Remote mode connection failures

**Symptom**: `gateway.mode: remote` but Control UI can't connect to `gateway.remote.url`.

**Checklist**:

1. **Verify remote URL is reachable**:
	```
	curl -k "$(openclaw config get gateway.remote.url)/health" \
	  -H "Authorization: Bearer $(openclaw config get gateway.remote.token)"
	```
2. **Check token/password**:
	```
	openclaw config get gateway.remote.token
	# Must match gateway.auth.token on remote host
	```
3. **Test WebSocket connection**:
	```
	openclaw gateway call health --verbose
	# Shows connection URL + auth mode
	```

**Common failures**:

- **SSH tunnel not active**: Ensure `ssh -N -L 18789:127.0.0.1:18789 user@host` is running
- **Tailscale Serve not running**: On remote host: `openclaw gateway --tailscale serve`
- **Firewall blocking**: Check remote host firewall (ufw/iptables/Security Groups)

**Code path**: Remote URL resolution in [src/gateway/call.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/gateway/call.ts#LNaN-LNaN)

**Sources**: [docs/gateway/remote.md](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/remote.md) [docs/help/faq.md 164-165](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L164-L165)

### OAuth token expired

**Symptom**: Model requests fail with "401 Unauthorized" or "expired\_token".

**Detection**:

```
openclaw models status     # Shows auth mode + expiry
openclaw doctor           # Warns about expiring tokens
```

**Provider-specific fixes**:

| Provider | Fix |
| --- | --- |
| **Anthropic** | Run `claude setup-token` elsewhere, then:   `openclaw models auth paste-token --provider anthropic` |
| **OpenAI Codex** | `openclaw models auth openai-codex`   (device flow re-auth) |
| **Google Gemini** | `openclaw models auth google-gemini-cli`   (OAuth flow) |
| **GitHub Copilot** | Set `COPILOT_GITHUB_TOKEN` or `GH_TOKEN`   (`gh auth token` if using GitHub CLI) |

**Code path**: Doctor checks expiry in [src/commands/doctor-auth.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/commands/doctor-auth.ts#LNaN-LNaN) refresh logic in [src/agents/auth-profiles.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/agents/auth-profiles.ts)

**Symptom**: Error message: `No credentials found for profile "anthropic:default"` or similar.

**Diagnosis**:

```
# List auth profiles
openclaw models status

# Check profile order
openclaw config get auth.order

# Inspect auth store (macOS)
openclaw config get-path  # Note OPENCLAW_STATE_DIR
```

1. **Profile not created**: Run auth flow for the provider:

## Channel-Specific Issues

### WhatsApp login/pairing failures

**Symptom**: QR code not appearing, or WhatsApp disconnects repeatedly.

**Diagnosis**:

```
openclaw channels status whatsapp
openclaw logs --follow | grep -i whatsapp
```

**Common causes**:

1. **WhatsApp Web already paired on 5 devices**: Unlink old sessions in WhatsApp mobile → Linked Devices.
2. **Auth state corrupted**: Remove auth state and re-pair:
	```
	rm -rf ~/.openclaw/credentials/whatsapp/default/*
	openclaw gateway restart
	# Scan new QR code
	```
3. **Outdated Baileys version**: Update OpenClaw:
	```
	openclaw update
	openclaw gateway restart
	```

**State directory** (default): `~/.openclaw/credentials/whatsapp/<accountId>/`

**Code path**: WhatsApp adapter in [src/channels/whatsapp/](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/channels/whatsapp/) auth state migration in [src/commands/doctor-state-migrations.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/commands/doctor-state-migrations.ts)

### systemd user service stops on logout

**Symptom** (Linux): Gateway stops working after SSH logout or idle timeout.

**Cause**: systemd user services require "lingering" to survive logout.

**Auto-fix**: `openclaw doctor` prompts to enable lingering when it detects a loaded user service.
