This page documents all built-in slash commands available in OpenClaw. Commands are invoked through chat messages starting with `/` and provide direct access to agent controls, session management, and system configuration. For command system architecture and handling, see [Commands and Directives](https://deepwiki.com/openclaw/openclaw/9-commands-and-directives). For platform-specific native command implementations, see [Platform-Specific Commands](https://deepwiki.com/openclaw/openclaw/9.2-platform-specific-commands). For inline directives (`/think`, `/verbose`, `/model`, etc.), see [Directives](https://deepwiki.com/openclaw/openclaw/9.3-directives).

## Command Types

OpenClaw supports two command surfaces:

| Type | Description | Example Surfaces |
| --- | --- | --- |
| **Text commands** | Parsed from message text starting with `/` | WhatsApp, Signal, iMessage, WebChat |
| **Native commands** | Platform-specific slash command APIs | Discord, Telegram, Slack |

Text commands work everywhere. Native commands provide enhanced UX (autocomplete, argument menus, ephemeral responses) on supported platforms but require explicit registration.

## Session Commands

### /new \[model\]

**Aliases**: `/reset`

Resets the session context and starts fresh. Optional `model` parameter accepts provider/model names or aliases.

**Behavior**:

- Clears conversation history
- Resets context tokens to 0
- Preserves session settings (think level, verbose, etc.)
- If `model` is provided and matches a known model, switches to it
- If `model` text doesn't match, treats remainder as message body

**Examples**:

```
/new
/reset
/new gpt-5.2
/new anthropic/claude-opus-4-5
```

Sources: [src/auto-reply/commands-registry.data.ts 195-203](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/auto-reply/commands-registry.data.ts#L195-L203) [docs/tools/slash-commands.md 82-100](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/tools/slash-commands.md#L82-L100)

### /compact \[instructions\]

**Scope**: Text only (not available as native command)

Manually triggers session context compaction. Optional `instructions` parameter provides guidance to the compaction process.

**Behavior**:

- Summarizes conversation history to reduce context usage
- Preserves important facts and decisions
- Returns summary of what was compacted
- Instructions are passed to the summarization model if provided

**Examples**:

```
/compact
/compact preserve the bug fix discussion
```

### /stop

Aborts the current agent run. The session remains active and can receive new messages immediately.

**Behavior**:

- Sends abort signal to running agent
- Stops tool execution mid-stream
- Returns "Agent was aborted" message
- Does not reset session state

## Options Commands

### /think <level>

**Aliases**: `/thinking`, `/t`

Sets the thinking level for models that support extended reasoning (GPT-5.2, Codex models).

**Arguments**:

- `level` (required): One of `off`, `minimal`, `low`, `medium`, `high`, `xhigh`

**Behavior**:

- When sent alone (directive-only), persists to session
- When inline with message text, applies to current turn only
- Available levels vary by model/provider
- Higher levels increase latency and token usage
- Returns acknowledgment with new level

**Examples**:

```
/think medium
/thinking high
/t off
```

### /model <name>

**Aliases**: `/models`

Switches the model for the current session or displays model picker.

**Arguments**:

- `name` (optional): Model identifier, alias, or provider name

**Formats**:


**Behavior**:

- Directive-only: persists model override to session
- Inline: applies to current turn only (future: may become directive-only)
- Picker shows model families with available providers
- Status view includes configured baseUrl and API mode

**Aliases**: `/v`

**Modes**:

| Mode | Description |
| --- | --- |
| `off` | Normal output (default) |

**Security Note**: In group chats, verbose output may reveal internal reasoning or tool output you didn't intend to share. Keep off in groups unless explicitly needed.

## Management Commands

### /config <action> \[path\] \[value\]

**Requires**: `commands.config: true` in config

Persists configuration changes to `openclaw.json`.

**Actions**:

| Action | Arguments | Description |
| --- | --- | --- |
| `show` | `[path]` | Display config (all or specific path) |
| `get` | `<path>` | Get a single value |
| `set` | `<path> <value>` | Set a value (JSON5 or string) |
| `unset` | `<path>` | Remove a value |

**Path Format**: Dot or bracket notation (e.g., `agents.defaults.model`, `channels.whatsapp.allowFrom[0]`)

**Examples**:

```
/config show
/config get messages.responsePrefix
/config set messages.responsePrefix="[openclaw]"
/config set channels.whatsapp.allowFrom=["+1555"]
/config unset messages.responsePrefix
```

**Notes**:

- Validates before write (rejects invalid changes)
- Persists across restarts
- Owner-only by default
- Honors `configWrites` policy per channel

### /tts <action> \[value\]

Controls Text-to-Speech behavior.

**Actions**:

| Action | Arguments | Description |
| --- | --- | --- |
| (none) | \- | Show current TTS status |
| `off` | \- | Disable TTS |
| `always` | \- | Enable for all messages |
| `inbound` | \- | Enable for user messages only |
| `tagged` | \- | Enable for messages with TTS tag |
| `status` | \- | Show detailed status |

| `limit` | `<n>` | Set max message length |
| `summary` | `on\|off` | Enable/disable summarization |
| `audio` | \- | Request audio attachment (one-time) |

**Platform Notes**:

## Command Authorization

### Access Control Flow

```
Command HandlerAuthorization Checkcommand detectedtruefalseInbound Message
text, sender, channelhasControlCommand()
or isControlCommandMessage()resolveCommandAuthorizedFromAuthorizers()
allowlist, pairing, dmPolicyCommandAuthorized: true/false
in MsgContextHandler checks
isAuthorizedSenderExecute command
return replyLog + silent drop
or treat as plain text
```

**Command Authorization Check**

Commands are only executed for authorized senders. Authorization is determined by:

1. **Allowlists**: `channels.<channel>.allowFrom` for DMs, `channels.<channel>.groups[*].allowFrom` for groups
2. **Pairing**: `dmPolicy: "pairing"` requires explicit pairing approval
3. **Access Groups**: `commands.useAccessGroups: true` enforces policies

**Inline Shortcuts Exception**: Certain commands (`/help`, `/commands`, `/status`, `/whoami`) work inline (e.g., `hey /status`) and are stripped before the model sees the text. Only authorized senders can use inline shortcuts.

**Unauthorized Behavior**:

- Command-only messages: silently dropped
- Inline directives: treated as plain text (passed to model)

## Fast Path Optimization

Command-only messages from authorized senders bypass the agent queue and are handled immediately:

**Criteria**:

- Message is command-only (no other text)
- Sender is authorized (allowlist or paired)
- Command is recognized

**Benefits**:

- Instant status checks (`/status`, `/whoami`)
- Immediate control actions (`/stop`, `/restart`)
- No queue waiting

**Exceptions**:

- Messages with both commands and text go through normal flow
- Unauthorized senders never use fast path
