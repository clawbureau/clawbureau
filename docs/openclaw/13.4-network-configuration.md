## Network Binding Modes

The Gateway supports five binding modes controlled by `gateway.bind`. The bind mode determines which network interfaces the Gateway listens on and which clients can connect.

### Configuration Precedence

```
--bind flag > OPENCLAW_GATEWAY_BIND > gateway.bind > default (loopback)
```

### Available Modes

| Mode | Interface | Auth Required | Use Case |
| --- | --- | --- | --- |
| `loopback` | `127.0.0.1` | Optional (recommended) | Local-only access; use with SSH tunnel or Tailscale Serve |
| `lan` | `0.0.0.0` | **Required** | LAN/VPN access; binds all interfaces |
| `tailnet` | Tailscale IP | **Required** | Direct Tailscale network access |
| `auto` | Detects available | Depends on interface | Fallback logic: loopback → tailnet → lan |
| `custom` | Config-specified | **Required** | Advanced: bind to specific IP via `gateway.bindAddress` |

**Sources:**[docs/help/faq.md 95-96](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L95-L96) [docs/gateway/troubleshooting.md 196-220](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L196-L220) [docs/gateway/index.md 29-30](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L29-L30)

---

Loopback binding (`gateway.bind: "loopback"`) restricts the Gateway to `127.0.0.1`, allowing connections only from the same host. This is the most secure option for production.

### Configuration

```
{
  "gateway": {
    "bind": "loopback",
    "port": 18789,
    "auth": {
      "mode": "token",
      "token": "your-secure-token-here"
    }
  }
}
```

### Remote Access Options

When using loopback, remote clients must connect via:

1. **SSH tunnel** (port forwarding)
2. **Tailscale Serve** (HTTPS with Tailscale identity auth)
3. Reverse proxy (nginx, Caddy)

**Loopback + SSH Tunnel**

```
ssh user@host -L 18789:127.0.0.1:18789Forwarded to localhost:18789TunnelClient
(laptop)SSH Tunnel
-L 18789:127.0.0.1:18789VPS/Server
(127.0.0.1:18789)Gateway
bind: loopback
```

## LAN Binding (0.0.0.0)

LAN binding (`gateway.bind: "lan"`) listens on all network interfaces (`0.0.0.0`), allowing connections from any device that can reach the host.

### Security Requirements

**Authentication is mandatory for LAN binding.** The Gateway refuses to start if `gateway.bind: "lan"` is set without `gateway.auth.token` or `gateway.auth.password`.

### Configuration

```
{
  "gateway": {
    "bind": "lan",
    "port": 18789,
    "auth": {
      "mode": "token",
      "token": "your-secure-token-here"
    }
  }
}
```

### Network Flow

```
Gateway HostLocal Network / VPNws://192.168.1.50:18789
token requiredws://192.168.1.50:18789
token requiredws://192.168.1.50:18789
token requiredLaptop
192.168.1.100Phone
192.168.1.101Tablet
192.168.1.1020.0.0.0:18789
(all interfaces)Gateway Process
Auth: token/password
```

## Tailnet Binding

Tailscale integration provides two modes: **Serve** (loopback + Tailscale reverse proxy) and **tailnet binding** (direct Tailscale IP binding).

Tailscale Serve keeps the Gateway on loopback (`127.0.0.1`) and exposes it via Tailscale's HTTPS reverse proxy with built-in identity authentication.

**Configuration**

```
{
  "gateway": {
    "bind": "loopback",
    "port": 18789,
    "auth": {
      "mode": "token",
      "allowTailscale": true  // Allow Tailscale identity headers
    },
    "tailscale": {
      "mode": "serve",
      "target": "https://localhost:18789"
    }
  }
}
```

**Starting Tailscale Serve**

```
openclaw gateway --tailscale serve
```

This runs `tailscale serve` in the background and configures the Gateway to accept Tailscale identity headers. Clients can then access `https://<magicdns>/` without a token (authenticated via Tailscale identity).

**Tailscale Serve Architecture**

```
https://hostname.ts.net
Tailscale identityhttp://127.0.0.1:18789
Funnelclient-Identity headerClient
(any tailnet device)Tailscale Serve
(HTTPS reverse proxy)127.0.0.1:18789
(Gateway)
```

**Sources:**[docs/help/faq.md 328-333](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L328-L333) [docs/platforms/oracle.md 109-149](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/platforms/oracle.md#L109-L149)

### Tailnet Direct Binding

Direct binding (`gateway.bind: "tailnet"`) binds the Gateway to the host's Tailscale IP (100.64.0.0/10 range). This requires auth and does not use Tailscale Serve.

**Configuration**

```
{
  "gateway": {
    "bind": "tailnet",
    "port": 18789,
    "auth": {
      "mode": "token",
      "token": "your-secure-token-here"
    }
  }
}
```

The Gateway will detect the Tailscale IP at startup and bind to it. If no Tailscale interface is found, startup fails.

**Sources:**[docs/gateway/troubleshooting.md 213-218](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L213-L218) [docs/help/faq.md 328-333](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L328-L333)

## Port Configuration

The Gateway uses two ports by default:

| Port | Purpose | Config Key | Default |
| --- | --- | --- | --- |
| Gateway | WebSocket + HTTP (multiplex) | `gateway.port` | `18789` |
| Canvas Host | A2UI file server | `canvasHost.port` | `18793` |

### Gateway Port

The Gateway port serves:

- WebSocket control plane (RPC protocol)
- HTTP endpoints (`/v1/chat/completions`, `/tools/invoke`, `/setup`)
- Control UI static assets

**Port Precedence**

```
--port flag > OPENCLAW_GATEWAY_PORT > gateway.port > 18789
```

**Sources:**[docs/gateway/index.md 29-35](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L29-L35)

### Canvas Host Port

The Canvas Host serves files from `~/.openclaw/workspace/canvas/` at `http://<gateway-host>:<canvasHost.port>/__openclaw__/canvas/`.

**Configuration**

```
{
  "canvasHost": {
    "enabled": true,
    "port": 18793,
    "bind": "loopback"  // Or match gateway.bind
  }
}
```

**Disabling Canvas Host**

```
export OPENCLAW_SKIP_CANVAS_HOST=1
openclaw gateway
```

Or set `canvasHost.enabled: false` in config.

**Sources:**[docs/gateway/index.md 34-35](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L34-L35) [docs/gateway/index.md 90-95](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L90-L95)

The `--dev` flag shifts all ports to avoid conflicts:

| Service | Default | Dev (`--dev`) |
| --- | --- | --- |
| Gateway | 18789 | 19001 |
| Browser control | 18791 | 19003 |
| Canvas host | 18793 | 19005 |

## Authentication Requirements

Authentication protects the Gateway from unauthorized access. The wizard generates a token by default, even for loopback binds.

### Token Authentication

Token auth uses `gateway.auth.token` (or `OPENCLAW_GATEWAY_TOKEN`). Clients include the token in `connect.params.auth.token`.

**Configuration**

```
{
  "gateway": {
    "auth": {
      "mode": "token",
      "token": "your-secure-random-token",
      "allowTailscale": false  // Require token even over Tailscale
    }
  }
}
```

**Client Connection (JavaScript)**

```
const ws = new WebSocket('ws://127.0.0.1:18789/');
ws.send(JSON.stringify({
  type: 'req',
  id: 'connect-1',
  method: 'connect',
  params: {
    minProtocol: 1,
    maxProtocol: 1,
    client: { id: 'my-client', displayName: 'My Client', version: '1.0.0', platform: 'web', mode: 'app' },
    auth: { token: 'your-secure-random-token' }
  }
}));
```

**Sources:**[docs/gateway/index.md 40-42](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L40-L42) [docs/start/wizard.md 114-117](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/start/wizard.md#L114-L117)

### Password Authentication

Password auth uses `gateway.auth.password` (or `OPENCLAW_GATEWAY_PASSWORD`). Clients include the password in `connect.params.auth.password`.

**Configuration**

```
{
  "gateway": {
    "auth": {
      "mode": "password",
      "password": "your-secure-password"
    }
  }
}
```

**Sources:**[docs/gateway/index.md 40-42](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L40-L42)

### Tailscale Identity Authentication

When `gateway.auth.allowTailscale: true` and the Gateway is accessed via Tailscale Serve, the `Funnelclient-Identity` header (injected by Tailscale) satisfies auth without a token.

**Configuration**

```
{
  "gateway": {
    "auth": {
      "mode": "token",
      "allowTailscale": true
    },
    "tailscale": {
      "mode": "serve"
    }
  }
}
```

**Sources:**[docs/help/faq.md 328-333](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L328-L333) [docs/platforms/oracle.md 109-149](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/platforms/oracle.md#L109-L149)

## SSH Tunnels

SSH tunnels forward the Gateway port from a remote host to your local machine, allowing secure access without exposing the Gateway to the internet.

### Local Port Forwarding

**Basic Tunnel**

```
ssh -N -L 18789:127.0.0.1:18789 user@remote-host
```

This forwards local port `18789` to the remote Gateway's `127.0.0.1:18789`. Clients can then connect to `ws://127.0.0.1:18789/` on the local machine.

**Multiple Ports (Gateway + Canvas)**

```
ssh -N -L 18789:127.0.0.1:18789 -L 18793:127.0.0.1:18793 user@remote-host
```

**Background Tunnel (macOS/Linux)**

```
ssh -f -N -L 18789:127.0.0.1:18789 user@remote-host
```

**SSH Tunnel Architecture**

```
ws://localhost:18789Tunnelssh -L 18789:127.0.0.1:18789Local Client
laptoplocalhost:18789
(forwarded)SSH TunnelRemote Host
(VPS)127.0.0.1:18789
(Gateway)
```

**Sources:**[docs/gateway/index.md 46-50](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L46-L50) [docs/platforms/digitalocean.md 103-110](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/platforms/digitalocean.md#L103-L110)

### Windows Port Forwarding (WSL2 to LAN)

WSL2 uses a virtual network, so external devices cannot reach services inside WSL by default. Use `netsh portproxy` to forward a Windows port to the WSL IP.

**Example (PowerShell as Administrator)**

```
$WslIp = (wsl -- hostname -I).Trim().Split(" ")[0]
netsh interface portproxy add v4tov4 listenaddress=0.0.0.0 listenport=18789 \`
  connectaddress=$WslIp connectport=18789
```

**Firewall Rule**

```
New-NetFirewallRule -DisplayName "OpenClaw Gateway" -Direction Inbound \`
  -Protocol TCP -LocalPort 18789 -Action Allow
```

**Sources:**[docs/platforms/windows.md 54-96](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/platforms/windows.md#L54-L96)

## Security Best Practices

### Bind Mode Selection

| Scenario | Recommended Bind | Rationale |
| --- | --- | --- |
| Local development | `loopback` | No external exposure |
| VPS with SSH access | `loopback` + SSH tunnel | Minimal attack surface |
| VPS with Tailscale | `loopback` + Tailscale Serve | HTTPS + identity auth |
| LAN access (home network) | `lan` + strong token | Convenience with auth |
| Public internet | **Not recommended** | Use reverse proxy + HTTPS instead |

**Sources:**[docs/gateway/index.md 46-50](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L46-L50) [docs/help/faq.md 328-333](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L328-L333)

(recommended)Auth: REQUIRED

**Sources:**[docs/gateway/troubleshooting.md 196-220](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L196-L220) [docs/gateway/index.md 40-42](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L40-L42)

### Token/Password Strength

**Token Generation (recommended)**

```
# Generate a secure 32-byte token
openssl rand -hex 32
```

**Config Storage**

Tokens can be stored in:

1. `gateway.auth.token` (in `openclaw.json`)
2. `OPENCLAW_GATEWAY_TOKEN` (in `~/.openclaw/.env`)

Environment variables take precedence over config file values.

## Troubleshooting

### Gateway Won't Start: "refusing to bind without auth"

**Symptom:** Gateway exits immediately with a bind/auth error.

**Cause:** Non-loopback bind (`lan`, `tailnet`) without `gateway.auth.token` or `gateway.auth.password`.

**Fix:**

```
openclaw config set gateway.auth.mode token
openclaw config set gateway.auth.token "$(openssl rand -hex 32)"
openclaw gateway restart
```

**Sources:**[docs/gateway/troubleshooting.md 211-214](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L211-L214)

### Control UI Shows "Unauthorized"

**Symptom:** Control UI connects but shows "unauthorized" banner.

**Cause:** Client did not send the required token/password.

**Fix:**

1. Run `openclaw dashboard` to get the tokenized URL
2. Or manually append `?token=your-token` to the URL
3. Or paste the token into Control UI settings (stored in browser)

**Sources:**[docs/help/faq.md 328-333](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L328-L333) [docs/gateway/troubleshooting.md 75-87](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L75-L87)

### Tailnet Bind Fails: "no tailnet interface found"

**Symptom:** Gateway exits with "no tailnet interface found" when using `bind: "tailnet"`.

**Cause:** Tailscale is not running or no Tailscale IP is assigned.

**Fix:**

```
# Check Tailscale status
tailscale status

# If not running, start it
tailscale up

# Verify IP is in 100.64.0.0/10 range
ip addr show
```

**Sources:**[docs/gateway/troubleshooting.md 215-218](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L215-L218)

### RPC Probe Fails but Service is Running

**Symptom:**`openclaw gateway status` shows "Runtime: running" but "RPC probe: failed".

**Cause:** Gateway bound to non-loopback interface, but CLI is probing `127.0.0.1`.

**Fix:**

```
# Check the actual bind mode
openclaw gateway status

# Use --url to probe the correct address
openclaw gateway status --url ws://192.168.1.50:18789
```

**Sources:**[docs/gateway/troubleshooting.md 189-225](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L189-L225)

| --- | --- | --- | --- | --- |
| Dev (local) | Development | `loopback` | Optional | `http://127.0.0.1:18789` |
| VPS + SSH | Production | `loopback` | Recommended | SSH tunnel + token |
| VPS + Tailscale Serve | Production | `loopback` | Tailscale identity | `https://hostname.ts.net` |
| VPS + Tailscale Direct | Production | `tailnet` | Required | `ws://100.64.x.x:18789` |
| Home LAN | Convenience | `lan` | Required | `ws://192.168.x.x:18789` |

**Key Takeaways:**

1. **Default is secure:** Loopback bind + token auth (even for local dev)
2. **Non-loopback requires auth:** Gateway refuses to start without token/password
3. **Tailscale Serve is easiest remote option:** HTTPS + identity auth, no port forwarding
4. **SSH tunnels are universal:** Work everywhere, no additional software needed
