This page documents the different methods for installing OpenClaw and compares their trade-offs. It focuses on **how** each installation method works and **when** to choose one over another.

The installer script is the recommended installation method for most users. It handles platform detection, dependency installation, and runs onboarding.

### How It Works

The installer script performs these steps:

1. **Platform detection** - Identifies macOS, Linux, or WSL2
2. **Node.js installation** - Ensures Node 22+ via Homebrew (macOS) or NodeSource (Linux)
3. **npm PATH fix** - Reconfigures npm prefix to `~/.npm-global` on Linux when needed
4. **Install method selection** - Chooses between npm global install or git checkout
5. **Package installation** - Runs `npm install -g openclaw@latest` or git clone + build
6. **Post-install repair** - Runs `openclaw doctor --non-interactive` on upgrades
7. **Onboarding** - Optionally launches `openclaw onboard`

**Sources:**[docs/install/installer.md 33-73](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/installer.md#L33-L73)

### Sharp Native Build Workaround

The `sharp` image processing library can cause install failures when a global libvips is detected. The installer mitigates this by setting `SHARP_IGNORE_GLOBAL_LIBVIPS=1`, which forces prebuilt binaries.

```
# If sharp fails to install
SHARP_IGNORE_GLOBAL_LIBVIPS=1 npm install -g openclaw@latest
```

**Sources:**[docs/install/index.md 68-74](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/index.md#L68-L74) [docs/install/installer.md 44-49](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/installer.md#L44-L49)

## Update Mechanisms

Each installation method has a different update mechanism:

| Method | Update Command | What It Does | Config Preservation |
| --- | --- | --- | --- |
| **Installer (npm)** | `openclaw update` or `npm update -g openclaw` | Fetches latest from npm registry | Preserves `~/.openclaw/` |
| **Installer (git)** | `openclaw update --channel dev` or `git pull` + rebuild | Pulls from GitHub, rebuilds | Preserves `~/.openclaw/` |
| **Manual npm** | `npm update -g openclaw` or `npm install -g openclaw@<version>` | Standard npm update | Preserves `~/.openclaw/` |
| **Git/Source** | `git pull` + `pnpm install` + `pnpm build` | Manual rebuild | Preserves `~/.openclaw/` |
| **Docker** | `docker compose pull` + `docker compose up -d` or rebuild | Re-pull/rebuild image | Preserves mounted volumes |
| **macOS App** | Sparkle auto-update (Check for Updates menu) | Downloads and installs signed app update | Preserves `~/.openclaw/` |

**Sources:**[docs/install/updating.md](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/updating.md) [docs/help/faq.md 584-608](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L584-L608)

## Installation Method Selection Criteria

### When to Use Installer Script

- **First-time users** who want the recommended path
- **Systems where you have sudo** for Node/Homebrew install
- **Automated setups** (CI, provisioning scripts) with `--no-prompt`
- **You want onboarding to run** automatically

**Sources:**[docs/install/index.md 36-58](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/index.md#L36-L58) [docs/help/faq.md 293-316](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L293-L316)

### When to Use Manual npm

- **You already have Node 22+** and npm configured
- **You want fine-grained control** over npm flags (e.g., custom registry)
- **CI/CD pipelines** where you manage dependencies explicitly
- **You're familiar with npm** and prefer standard workflows

**Sources:**[docs/install/index.md 62-86](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/index.md#L62-L86)

### When to Use Git/Source Install

- **Contributors** developing features or fixing bugs
- **Plugin authors** who need to test against local changes
- **You need to modify source code** (custom skills, patches)
- **Debugging deep issues** where you need full visibility
- **Hackable setup** recommended by FAQ for stuck users with local AI agents

**Sources:**[docs/install/index.md 88-98](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/index.md#L88-L98) [docs/help/faq.md 247-274](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L247-L274)

### When to Use Docker

- **Cloud deployments** (Fly.io, GCP, Hetzner, Railway)
- **You want isolation** from the host system
- **Reproducible environments** across dev/staging/production
- **Multi-tenant** or multi-instance setups on the same host

**Sources:**[docs/platforms/fly.md 1-466](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/platforms/fly.md#L1-L466) [docs/platforms/gcp.md 1-499](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/platforms/gcp.md#L1-L499)

### When to Use macOS App

- **macOS users** who prefer a native app over CLI
- **Menu bar integration** is desired
- **Auto-updates via Sparkle** are preferred
- **No terminal** interaction wanted for basic operations

**Sources:**[docs/platforms/mac/release.md 1-78](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/platforms/mac/release.md#L1-L78)

## Common Installation Issues

### Issue: openclaw: command not found

**Cause:** npm global bin directory not on PATH.

**Diagnosis:**

```
npm prefix -g
echo "$PATH"
```

**Fix:** Add `$(npm prefix -g)/bin` to your shell's PATH.

**Sources:**[docs/install/index.md 153-176](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/index.md#L153-L176) [docs/help/troubleshooting.md 30-36](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/troubleshooting.md#L30-L36)

### Issue: sharp build failures

**Cause:** npm tries to build sharp against system libvips.

**Fix:**

```
SHARP_IGNORE_GLOBAL_LIBVIPS=1 npm install -g openclaw@latest
```

**Sources:**[docs/install/index.md 68-74](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/index.md#L68-L74) [docs/install/installer.md 44-49](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/installer.md#L44-L49)

**Cause:** Git for Windows not installed or not on PATH.

**Fix:** Install Git for Windows from [https://git-scm.com/download/win](https://git-scm.com/download/win)

**Sources:**[docs/help/faq.md 518-537](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L518-L537) [docs/install/installer.md 117-120](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/installer.md#L117-L120)

### Issue: Linux EACCES on global install

**Cause:** npm prefix points to root-owned directory.

**Fix:** The installer script reconfigures npm prefix to `~/.npm-global`. Manually:

```
npm config set prefix ~/.npm-global
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

**Sources:**[docs/install/installer.md 65-72](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/install/installer.md#L65-L72)

### Commit Hash Resolution Priority

1. `GIT_COMMIT` or `GIT_SHA` environment variables
2. `dist/build-info.json` (from build process)
3. `package.json` `gitHead` field (npm packages)
4. `.git/HEAD` (git checkouts)
