For detailed installation options and configuration, see [Installation](https://deepwiki.com/openclaw/openclaw/2-installation). For a comprehensive beginner guide, see Getting Started documentation. For understanding core concepts before starting, see [Key Concepts](https://deepwiki.com/openclaw/openclaw/1.1-key-concepts).

The quickest path to your first conversation is through the Control UI (no channel setup required): run `openclaw dashboard` or open `http://127.0.0.1:18789/` in your browser after the Gateway is running.

## Prerequisites

**Required:**

- Node.js ≥ 22 (Gateway requires this runtime)
- Internet connection for model API access

**Recommended:**

- `pnpm` for package management (if building from source)
- Terminal with modern ANSI color support

**Note on Bun:** The Gateway should not be run with Bun when using WhatsApp or Telegram channels due to compatibility issues. Use Node.js as the runtime.

## Onboarding Wizard

The onboarding wizard (`openclaw onboard`) configures your Gateway, workspace, model authentication, and channels in one guided flow.

### Running the Wizard

```
openclaw onboard --install-daemon
```

The `--install-daemon` flag installs the Gateway as a background service (LaunchAgent on macOS, systemd on Linux).

### What the Wizard Configures

**QuickStart vs Advanced Mode:** The wizard offers two paths:

- **QuickStart:** Uses sensible defaults (loopback binding, port 18789, token auth, Telegram/WhatsApp with allowlist)
- **Advanced:** Prompts for every configuration option

**Configuration Steps:**

1. **Model/Auth Setup:** Choose from:
	- Anthropic API key (recommended for Claude models)
	- Anthropic setup-token (from `claude setup-token` CLI)
	- OpenAI Codex subscription (OAuth flow)
	- OpenAI API key
	- Other providers (MiniMax, Moonshot, OpenRouter, etc.)
2. **Workspace Location:** Default `~/.openclaw/workspace` (seeds bootstrap files like `AGENTS.md`, `SOUL.md`)
3. **Gateway Settings:**
	- Port (default 18789)
	- Bind mode (loopback/lan/tailnet)
	- Auth mode (token recommended, even for loopback)
	- Tailscale exposure (off/serve/funnel)
4. **Channel Setup:** Optional configuration for messaging platforms (can be skipped and added later)
5. **Daemon Install:** Installs Gateway as a service with Node.js runtime
6. **Skills Setup:** Optional dependency installation for available skills

**Key Configuration Files Written:**

| Path | Purpose |
| --- | --- |
| `~/.openclaw/openclaw.json` | Main configuration file |
| `~/.openclaw/credentials/oauth.json` | OAuth credentials (legacy import) |
| `~/.openclaw/agents/<agentId>/agent/auth-profiles.json` | Auth profiles (API keys + OAuth) |
| `~/.openclaw/workspace/` | Agent workspace with bootstrap files |
| `~/.openclaw/.env` | Environment variables loaded by Gateway |

## Starting the Gateway

After onboarding with `--install-daemon`, the Gateway runs automatically as a background service. To check its status:

```
openclaw gateway status
```

**Manual Start (Foreground):**

If you need to run the Gateway in the foreground for debugging:

```
openclaw gateway --port 18789 --verbose
```

**Gateway Service Management:**

```
# Start service
openclaw gateway start

# Stop service
openclaw gateway stop

# Restart service
openclaw gateway restart
```

**Default Binding:** The Gateway binds to `ws://127.0.0.1:18789` by default (loopback-only). The same port also serves HTTP for:

- Control UI at `/`
- OpenAI-compatible Chat Completions at `/v1/chat/completions`
- Canvas file server (default port 18793)

## Connecting Your First Channel

### WhatsApp (QR Login)

The simplest channel to set up is WhatsApp via QR code:

```
openclaw channels login
```

This command:

1. Starts a temporary WhatsApp Web session
2. Displays a QR code in your terminal
3. Prompts you to scan it via WhatsApp → Settings → Linked Devices

Credentials are stored at `~/.openclaw/credentials/whatsapp/default/`.

### Telegram Bot

For Telegram, you need a bot token from [@BotFather](https://t.me/botfather):

```
openclaw channels add --channel telegram --token YOUR_BOT_TOKEN
```

Or add manually to `openclaw.json`:

```
{
  "channels": {
    "telegram": {
      "default": {
        "botToken": "YOUR_BOT_TOKEN"
      }
    }
  }
}
```

### Discord Bot

For Discord, create a bot at the [Discord Developer Portal](https://discord.com/developers/applications):

```
openclaw channels add --channel discord --token YOUR_BOT_TOKEN
```

**DM Security Default:** First-time DMs trigger pairing codes. Approve them via:

```
openclaw pairing list whatsapp
openclaw pairing approve whatsapp <code>
```

Or configure allowlists in `openclaw.json`:

```
{
  "channels": {
    "whatsapp": {
      "default": {
        "allowFrom": ["+15555550123"]
      }
    }
  }
}
```

## Verification

After setup, verify the system is working:

### Quick Health Check

```
openclaw status
```

This command shows:

- Gateway reachability
- Service status
- Agent configuration
- Channel connection status

### Deep Health Check

```
openclaw status --deep
```

This runs additional probes including provider auth checks.

### Test Message

Send a test message via CLI:

```
openclaw message send --target +15555550123 --message "Hello from OpenClaw"
```

### Common Issues

| Issue | Check | Fix |
| --- | --- | --- |
| Gateway not reachable | `openclaw gateway status` | `openclaw gateway restart` |
| No model auth | `openclaw models status` | Rerun `openclaw onboard` or `openclaw models auth add` |
| Channel not responding | `openclaw channels status` | Check credentials, run `openclaw doctor` |
| Config invalid | `openclaw doctor` | Apply suggested repairs |

## Doctor Command

The `openclaw doctor` command performs automated health checks and repairs:

```
openclaw doctor
```

**What it checks:**

- Config file validity and schema compliance
- Auth profile health (OAuth expiry, API key presence)
- Gateway service configuration
- Legacy state migrations
- Security warnings (permissions, bind modes)
- Sandbox image freshness
- Workspace integrity

**Headless mode:**

```
openclaw doctor --yes          # Accept all repairs
openclaw doctor --non-interactive  # Safe migrations only
```

The doctor command is implemented in [src/commands/doctor.ts 65-306](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/commands/doctor.ts#L65-L306) and delegates to specialized health check modules:

- `doctor-config-flow.ts` for config validation
- `doctor-auth.ts` for auth profile health
- `doctor-gateway-daemon-flow.ts` for service repairs
- `doctor-workspace-status.ts` for workspace checks

**Sources:**[src/commands/doctor.ts 65-306](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/commands/doctor.ts#L65-L306) [docs/gateway/doctor.md 9-51](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/doctor.md#L9-L51) [docs/help/faq.md 232-236](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L232-L236)

## Opening the Dashboard

Once the Gateway is running, open the browser Control UI:

**Local Access:**

```
http://127.0.0.1:18789/
```

If a token is configured, run:

```
openclaw dashboard
```

This opens your browser with a tokenized URL (`?token=...`).

**Remote Access Options:**

1. **Tailscale Serve (Recommended):** Exposes via HTTPS with identity-based auth
2. **SSH Tunnel:** Forward port 18789 through SSH
3. **LAN Binding:** Bind to `0.0.0.0` (requires strong token auth)

For remote access details, see the Remote Access documentation.

After getting OpenClaw running:

1. **Configure Additional Channels:** Add Telegram, Discord, Signal, or other platforms via `openclaw channels add` or `openclaw configure`
2. **Review Security Settings:** Run `openclaw security audit --deep` to check for security issues
3. **Customize Your Agent:** Edit workspace files:
	- `~/.openclaw/workspace/AGENTS.md` – System instructions
	- `~/.openclaw/workspace/SOUL.md` – Personality and behavior
	- `~/.openclaw/workspace/USER.md` – User information
	- `~/.openclaw/workspace/MEMORY.md` – Long-term memory
4. **Install Skills:** Use `openclaw skills list` to see available skills and `npx clawdhub` to browse and install additional skills
5. **Set Up Nodes:** Install the iOS or Android app to enable camera, location, and canvas capabilities
6. **Configure Automation:** Set up cron jobs with `openclaw cron add` or webhooks for external integrations
7. **Remote Access:** Configure Tailscale Serve or SSH tunneling for access when away from the Gateway host
