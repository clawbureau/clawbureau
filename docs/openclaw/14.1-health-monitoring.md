This page covers operational monitoring of OpenClaw: status commands, health checks, logging, and the presence system. These tools help diagnose runtime issues, verify Gateway reachability, and track client connections.

For troubleshooting specific failures, see [Common Issues](https://deepwiki.com/openclaw/openclaw/14.3-common-issues). For automated repairs, see [Doctor Command Guide](https://deepwiki.com/openclaw/openclaw/14.2-doctor-command-guide).

## Status Commands

OpenClaw provides several status commands with different scopes and depths. Use them in order when diagnosing issues.

### Quick Status (openclaw status)

The fastest local summary: OS info, update availability, Gateway reachability/mode, service state, agents/sessions, and provider configuration.

```
openclaw status
```

**What it reports:**

- Local system (OS, OpenClaw version, update availability)
- Gateway mode (`local` or `remote`) and reachability
- Service supervisor state (launchd/systemd/schtasks)
- Configured agents and active sessions
- Provider configuration state (does not probe live auth)

**When to use:** First diagnostic step for any issue.

**Sources:**[docs/help/faq.md 196-243](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L196-L243) [docs/cli/index.md 516-527](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/cli/index.md#L516-L527) [docs/gateway/troubleshooting.md 16-26](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L16-L26)

### Full Diagnosis (openclaw status --all)

Read-only, pasteable report including log tail. Tokens are redacted for safe sharing.

```
openclaw status --all
```

**Additional output:**

- Full config summary
- Recent log entries
- Environment variables (sensitive values masked)
- Service install metadata

**When to use:** When sharing a debug report or filing an issue.

**Sources:**[docs/help/faq.md 205-209](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L205-L209) [docs/gateway/troubleshooting.md 22](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L22-L22)

### Deep Probes (openclaw status --deep)

Requires a reachable Gateway. Runs live health checks including provider probes.

```
openclaw status --deep
```

**Additional checks:**

- Gateway health RPC (full snapshot)
- Provider auth probes (may consume tokens/trigger rate limits)
- Channel connectivity tests
- Node pairing status

**When to use:** When "configured" doesn't mean "working" — confirms live functionality.

**Sources:**[docs/help/faq.md 217-221](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L217-L221) [docs/gateway/troubleshooting.md 22](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L22-L22) [docs/start/wizard.md 139](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/start/wizard.md#L139-L139)

### Gateway Service Status (openclaw gateway status)

Supervisor-specific view: runtime state, PID, last exit code, and RPC probe result.

```
openclaw gateway status
```

**What it shows:**

- Supervisor runtime state (running/stopped/crashed)
- Process PID (if running)
- Last exit code and timestamp (if exited)
- RPC probe result (can the CLI connect to the WebSocket?)
- Config path mismatches (CLI vs service)
- Last Gateway error from logs (when service appears running but port is closed)

**When to use:** Service looks "loaded" but nothing responds, or you see config mismatches.

**Sources:**[docs/gateway/troubleshooting.md 98-104](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L98-L104) [docs/cli/index.md 620-630](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/cli/index.md#L620-L630) [docs/help/faq.md 210-215](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L210-L215)

### Gateway Probe (openclaw gateway probe)

Discovery and reachability check for local and remote Gateways.

```
openclaw gateway probe
```

**What it does:**

- Discovers Gateways via Bonjour/Avahi (local network)
- Tests reachability of configured Gateway URL
- Reports probe target and config source

**When to use:** Suspect you're probing the wrong Gateway, or need to verify remote access.

**Sources:**[docs/gateway/troubleshooting.md 23](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L23-L23)

## Health Checks

### Health RPC (openclaw health)

Fetches a comprehensive health snapshot from the running Gateway via WebSocket RPC.

```
openclaw health
openclaw health --json
openclaw health --verbose
```

**Snapshot contents (matching [src/gateway/protocol/schema/snapshot.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/gateway/protocol/schema/snapshot.ts)):**

- **Presence:** Active client connections (WebSocket and recent activity)
- **Channels:** Configured channels and connection state
- **Agents:** Configured agents and workspace paths
- **Models:** Configured providers and default models (does not probe auth)
- **Sessions:** Active sessions per agent
- **System:** Uptime, version, platform, Gateway mode
- **State version:** Incremented on presence changes (clients use for delta detection)

**When to use:**

- Verify Gateway is reachable and responding
- Confirm agent/channel/model configuration
- Check active client connections (presence)
- Automated monitoring scripts (use `--json`)

**Sources:**[docs/cli/index.md 542-549](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/cli/index.md#L542-L549) [docs/gateway/index.md 126](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L126-L126) [docs/help/faq.md 240-244](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L240-L244)

### Health vs Status

| Command | Scope | Requires Gateway | Use Case |
| --- | --- | --- | --- |
| `openclaw status` | Local + Gateway reachability | No (reports unreachable) | First diagnostic check |
| `openclaw status --all` | Local + logs + config | No | Pasteable debug report |
| `openclaw status --deep` | Local + Gateway health + probes | Yes | Verify live functionality |
| `openclaw health` | Gateway snapshot only | Yes | Programmatic monitoring |
| `openclaw gateway status` | Service supervisor state | No | Service lifecycle debugging |

### RPC-Based Logs (openclaw logs)

Tail Gateway file logs via RPC. Supports structured/colorized output in TTY sessions.

```
openclaw logs --follow
openclaw logs --limit 200
openclaw logs --json
openclaw logs --plain
```

**Options:**

- `--follow`: Continuous tail (like `tail -f`)
- `--limit <n>`: Show last N lines (default 100)
- `--json`: Line-delimited JSON (one event per line)
- `--plain`: Disable ANSI colors
- `--no-color`: Also disables ANSI colors

**Behavior:**

- TTY sessions render colorized, structured view
- Non-TTY falls back to plain text
- `--json` emits raw JSONL for scripting

**Sources:**[docs/cli/index.md 633-648](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/cli/index.md#L633-L648) [docs/gateway/troubleshooting.md 106-112](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L106-L112)

### File Logs

Always written to disk, independent of service supervisor.

**Default path:**`/tmp/openclaw/openclaw-YYYY-MM-DD.log`

**Format:** JSONL (one JSON object per line)

**Configuration:**[src/config/schema.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/config/schema.ts) `logging.*` section

- `logging.level`: `"error"` | `"warn"` | `"info"` | `"debug"` | `"trace"` (default `"info"`)
- `logging.file`: Custom path (default `/tmp/openclaw/`)
- `logging.consoleLevel`: Separate level for stdout (default `"warn"`)
- `logging.consoleStyle`: `"pretty"` | `"json"` (default `"pretty"` in TTY)

**Direct access:**

```
tail -f "$(ls -t /tmp/openclaw/openclaw-*.log | head -1)"
```

**Sources:**[docs/gateway/troubleshooting.md 106-124](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L106-L124) [docs/help/faq.md 223-231](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L223-L231)

### Service Logs

When running as a service, stdout/stderr are captured by the supervisor.

**macOS (LaunchAgent):**

```
# Standard output
cat "$OPENCLAW_STATE_DIR/logs/gateway.log"

# Standard error
cat "$OPENCLAW_STATE_DIR/logs/gateway.err.log"
```

**Linux (systemd):**

```
journalctl --user -u openclaw-gateway.service -n 200 --no-pager
journalctl --user -u openclaw-gateway.service -f  # follow
```

**Windows (Task Scheduler):**

```
schtasks /Query /TN "OpenClaw Gateway" /V /FO LIST
```

**Sources:**[docs/gateway/troubleshooting.md 106-112](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L106-L112)

### Log Levels

Use `--verbose` for console output or increase `logging.level` for file logs:

```
{
  "logging": {
    "level": "debug",
    "consoleLevel": "debug",
    "consoleStyle": "pretty"
  }
}
```

**Important:**`--verbose` flag affects **console** output only. File logs are controlled by `logging.level`.

**Sources:**[docs/gateway/troubleshooting.md 113-123](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L113-L123) [docs/gateway/index.md 18-19](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L18-L19)

## Presence System

The presence system tracks active Gateway clients and recent activity. Presence entries are structured, deduplicated, and broadcast to all connected clients.

### What Is Presence?

Presence entries represent:

- **WebSocket clients:** Active connections (Control UI, TUI, macOS app, mobile apps, nodes)
- **Recent activity:** Commands, agent runs, system events (retained briefly after disconnect)

**Structure (from [src/gateway/protocol/schema/snapshot.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/gateway/protocol/schema/snapshot.ts)):**

```
{
  host: string;           // hostname or "unknown"
  ip?: string;           // client IP address
  version?: string;      // client version
  platform?: string;     // "macos" | "ios" | "linux" | "windows" | "android"
  deviceFamily?: string; // "Mac" | "iPhone" | "iPad" | etc.
  modelIdentifier?: string; // "MacBookPro18,3" | etc.
  mode?: string;         // "cli" | "app" | "control-ui" | "node"
  lastInputSeconds?: number; // seconds since last user input
  ts: number;            // timestamp (ms)
  reason?: string;       // human-readable status
  tags?: string[];       // additional metadata
  instanceId?: string;   // stable client identifier
}
```

**Sources:**[docs/gateway/index.md 122](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L122-L122) [docs/concepts/presence.md](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/concepts/presence.md) (see page #934 in table of contents)

### Presence RPC (system-presence)

Retrieve current presence list from the Gateway:

```
openclaw system presence
openclaw system presence --json
```

**Gateway RPC:**

```
// Request
{ type: "req", id: "...", method: "system-presence", params: {} }

// Response
{ type: "res", id: "...", ok: true, payload: PresenceEntry[] }
```

**Sources:**[docs/gateway/index.md 128](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L128-L128) [src/gateway/protocol/index.ts](https://github.com/openclaw/openclaw/blob/bf6ec64f/src/gateway/protocol/index.ts)

### Presence Updates (Events)

Presence changes are broadcast to all connected WebSocket clients as `event:"presence"` frames:

```
{
  type: "event",
  event: "presence",
  payload: PresenceEntry[],  // full presence list (snapshot)
  stateVersion: number       // incremented on each change
}
```

Clients detect gaps in `stateVersion` and re-fetch the full presence list when needed.

**Sources:**[docs/gateway/index.md 143-144](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L143-L144) [docs/gateway/index.md 174](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L174-L174)


**Sources:**[docs/gateway/index.md 137-175](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L137-L175) [docs/concepts/presence.md](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/concepts/presence.md)

### Use Cases for Presence Monitoring

1. **Multi-client coordination:** Identify which clients are connected and active
2. **Node availability:** Check if paired nodes are currently connected
3. **Activity tracking:** Detect idle clients (`lastInputSeconds`)
4. **Debugging disconnects:** Review recent presence entries for disconnect reasons
5. **Capacity planning:** Monitor concurrent client count

**Sources:**[docs/concepts/presence.md](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/concepts/presence.md)

## Usage Tracking

OpenClaw surfaces provider usage/quota when OAuth or API credentials are available. Data comes directly from provider usage endpoints (no estimates).

### Supported Providers

- Anthropic (API keys + OAuth)
- GitHub Copilot (OAuth)
- OpenAI Codex (OAuth)
- Gemini CLI / Antigravity (when plugin enabled)

If no matching credentials exist, usage is hidden.

## Monitoring Patterns

### Regular Health Checks

Periodic health checks for production/VPS deployments:

```
#!/bin/bash
# Cron: */5 * * * * /path/to/health-check.sh

openclaw health --json > /tmp/openclaw-health.json
if [ $? -ne 0 ]; then
  # Gateway unreachable
  echo "Gateway down at $(date)" >> /var/log/openclaw-alerts.log
  # Send alert (email, webhook, etc.)
fi
```

**Sources:**[docs/cli/index.md 542-549](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/cli/index.md#L542-L549)

### Log Tailing During Troubleshooting

Live log monitoring while reproducing an issue:

```
# Terminal 1: tail logs
openclaw logs --follow

# Terminal 2: reproduce the issue
openclaw agent --message "test" --deliver
```

**Sources:**[docs/gateway/troubleshooting.md 223-230](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L223-L230)

### Presence for Multi-Client Deployments

Track which clients are connected:

```
openclaw system presence --json | jq '.[] | {mode, platform, host, lastInputSeconds}'
```

Example output:

```
{"mode": "control-ui", "platform": "macos", "host": "macbook.local", "lastInputSeconds": 15}
{"mode": "node", "platform": "ios", "host": "iphone.local", "lastInputSeconds": 120}
{"mode": "cli", "platform": "linux", "host": "vps.example.com", "lastInputSeconds": 3}
```

**Sources:**[docs/gateway/index.md 128](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/index.md#L128-L128)

### Pasteable Reports for Support

When filing issues or asking for help, use the read-only, safe-to-share report:

```
openclaw status --all > openclaw-status.txt
```

This redacts tokens and includes log tail, config summary, and environment details.

**Sources:**[docs/help/faq.md 205-209](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/help/faq.md#L205-L209) [docs/gateway/troubleshooting.md 28](https://github.com/openclaw/openclaw/blob/bf6ec64f/docs/gateway/troubleshooting.md#L28-L28)
