## Codebase Patterns
- Clawbounties Assurance Review v1 already made compiled evidence visible in submission, requester/reviewer, arena, and manager surfaces; this roadmap should promote those signals into canonical decision policy rather than inventing a second parallel trust model.
- Authoritative compiled evidence must stay distinct from optional narrative or reference-only attachments; only authoritative compiled posture can improve approval automation.
- The likely implementation surface remains concentrated in `services/_archived/clawbounties/src/index.ts`, so execution should stay mostly single-lane even if prompt prep, review, and packaging run in parallel.
- Approval, arena desk, and manager autopilot should reuse one shared compiled-evidence evaluator so policy drift does not emerge between manual and automated closure paths.
- If humans override non-green compiled evidence, the override must be explicit and auditable; silent overrides defeat the point of surfacing authoritative compiled posture.
